HTML Form Reference Sheet

Form Tags

Tag Name Description Attributes Links
<form> Form Wrapper element for the entirety of the form. accept-charset / action / autocomplete / method / name / novalidate / rel / target MDN
W3Schools
<input> Input Input tag is used to collect the users information. Different type attribute values will change the way information is collected. value / readonly / disabled / size / maxlength / min / max / multiple / pattern / placeholder / required / step / autofocus / height / width / list / type
<label> Label Defines a label for several form elements. Useful for screen-reader users as it reads out loud when the inout element comes in focus.
<select> Defines a drop-down list for selection.
<textarea> A multi-line input field is defined.
<button> Defines a clickable button.
<fieldset> Used to group related data together.
<legend> Defines the caption for the field set.
<datalist> Specifies a list of pre-defined options for the input.
<output> Represents the result of a calculation
<options> Defines the options available to select from.

<form> Attributes

Attribute Name Description Reference
accept-charset="" Accept Character-Set specifies the character encodings that are to be used for the form submission. W3Schools
action="" Action Specifies where to send the form-data when a form is submitted W3Schools
autocomplete="" Auto Complete Set to "on"/"off". The browser "may"/"may not" auto complete the entries. W3Schools
method="" Method Assigns a HTTP verb to the HTTP request. W3Schools
name="" Name Specifies the name of the form. W3Schools
novalidate="" No Validate It specifies that the form-data (input) should not be validated when submitted if present. W3Schools
rel="" Relation Specifies the relationship between the current document and the linked document. W3Schools
target="" Target Specifies where to receive the response that is received after submiting the form. W3Schools

<input> Attributes

Attribute Name Description Reference
value Value Specifies an initial value for an input field. W3Schools
readonly Read Only The value of the input field cannot be changed and will be sent when submitting the form.
disabled Disabled The value of a disabled input field is unusable and un-clickable. This input will not be sent when submitting the form.
size Size Defines the visible width, in characters, of the input field.
maxlength Max Length Defines the maximum number f characters allowed in the input field
min / max Min / Max Specifies the minimum/maximum values for the input field.
Valid inputs: number, range, date, datetime-local, month, time and week.
multiple Multiple Specifies that a user is allowed to enter more than one value for the input field.
Valid inputs: email and file.
pattern Pattern Specifies a regular expression that the input field's value is checked against, when the form is submitted.
Use the global title attribute to describe the pattern to help the user.
About regular expressions.
placeholder Place Holder Specifies short a hint that describes the expected value of an input field.
Valid inputs: text, search, url, tel, email, and password.
required Required Specifies that an input field must be filled out before submitting the form.
Valid inputs: text, search, url, tel, email, password, date pickers, number, checkbox, radio, and file.
step Step Specifies the legal number intervals for an input field.
autofocus Auto Focus Specifies that an input field should automatically get focus when the page loads.
heigth / width Height / Width Specify the height and width of an <input type="image"> element.
list List Refers to a <datalist> element that contains pre-defined options for an <input> element.

<input type=""> attribute options

Options Description Reference
text Defines a single-line text input field. W3Schools
password Defines a password field
submit Defines a button for submitting form data to a form-handler.
reset Defines a reset button that will reset all form values to their default values
radio Defines a radio button.
Radio buttons let a user select ONLY ONE of a limited number of choices.
checkbox Defines a checkbox
Lets select zero or more options of a limited number.
button Defines a button
color Used for input fields that should contain a color.
Depending on browser support, a color picker can show up in the input field.
date Used for input fields that should contain a date.
datetime-local Specifies a date and time input field, with no time zone.
email Used for input fields that should contain an e-mail address.
file Defines a file-select field and a "Browse" button for file uploads.
month Allows the user to select a month and year.
number Defines a numeric input field.
range Defines a control for entering a number whose exact value is not important (like a slider control).
search Used for search fields.
tel Used for input fields that should contain a telephone number.
time Allows the user to select a time (no time zone).
url Used for input fields that should contain a URL address.
week Allows the user to select a week and year.
